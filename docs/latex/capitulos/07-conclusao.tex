\begin{resumo}
O projeto Poke-Adocao consiste num sistema de adoção de Pokémon que utiliza geolocalização para interligar os utilizadores a criaturas virtuais no espaço físico. A arquitetura baseia-se num monorepo que integra uma API RESTful em Python com FastAPI no backend, e uma aplicação móvel multiplataforma construída em React Native e Expo no frontend. O núcleo do processo de adoção é gerido por uma Máquina de Estados Finitos (FSM). Atualmente operando de forma binária para garantir a integridade do ciclo de vida restrito a um limite rigoroso de 50 metros de distância, a FSM demonstra a fundação arquitetural necessária para suportar, futuramente, estados intermediários mais dinâmicos e contratos \textit{player-to-player} (P2P) complexos, como transações pendentes de aprovação. Para viabilizar este mecanismo, o sistema implementa um motor espacial que calcula distâncias geográficas utilizando a fórmula de Haversine diretamente na camada da aplicação, processo previamente otimizado por uma filtragem através de \textit{Bounding Box}. A persistência de dados é assegurada de forma leve e contida por um banco de dados relacional SQLite, escolhido estrategicamente para validar as transações de \textit{optimistic locking} e o fluxo de ponta a ponta do protótipo sem a sobrecarga de configuração de infraestrutura externa. O presente documento detalha as decisões de engenharia, a infraestrutura e os requisitos técnicos que sustentam a aplicação.

\vspace{\onelineskip}
\noindent
\textbf{Palavras-chave:} Geolocalização. React Native. FastAPI. Máquina de Estados Finitos. Haversine. SQLite.
\end{resumo}

\begin{resumo}[Abstract]
\begin{otherlanguage*}{english}
The Poke-Adocao project is a Pokemon adoption system that leverages geolocation to connect users with virtual creatures in the physical world. The architecture is built upon a monorepo integrating a RESTful API developed in Python with FastAPI for the backend, and a cross-platform mobile application built with React Native and Expo for the frontend. The core adoption process is managed by a Finite State Machine (FSM). Currently operating in a binary manner to enforce the lifecycle under a strict 50-meter geographic threshold, the FSM demonstrates the architectural foundation necessary to support complex intermediate states and dynamic player-to-player (P2P) contracts in the future, such as pending approvals. To enable this mechanism, the system implements a spatial engine that computes geographic distances using the Haversine formula at the application layer, optimized by a Bounding Box pre-filtering process. Data persistence is handled in a lightweight and contained manner by a relational SQLite database, strategically chosen to validate optimistic locking transactions and the end-to-end flow of the prototype without the overhead of external infrastructure configuration. This document details the engineering decisions, infrastructure, and technical requirements underlying the application.

\vspace{\onelineskip}
\noindent
\textbf{Keywords:} Geolocation. React Native. FastAPI. Finite State Machine. Haversine. SQLite.
\end{otherlanguage*}
\end{resumo}
